# Science Disco Design Guidelines

## Introduction

These design guidelines ensure consistency and quality across all Discovery Day websites. Follow these standards when building your data visualizations and interactive pages. GitHub Copilot can help you implement these guidelines—simply reference this file in your prompts.

**How to use these guidelines:**
- Apply the brand colors and typography to all HTML pages
- Ensure your visualizations are clear and accessible
- Test your page on a laptop screen at full width
- Use these as a checklist before presenting your final website

---

## Design Standards

### Brand Colors
Use the Science Disco color palette consistently:
- **Primary pink:** `#D94B8A` (for headings, highlights, call-to-action elements)
- **Deep plum:** `#3D1A5C` (for dark text, background gradients)
- **Lavender:** `#C4A8D4` (for subtle accents, borders, secondary elements)
- **Teal:** `#5BBFBA` (for data visualization highlights, interactive elements)
- **Light background:** `#FDF4F9` (for content areas, cards, panels)

### Typography
- **Font family:** Montserrat (load from Google Fonts)
- **Headings:** Use Black (900) or ExtraBold (800) weights
- **Body text:** Use Regular (400) weight
- **Font sizes:** Ensure readability—minimum 16px for body text, scale headings appropriately

### Layout & Responsiveness
- The page must be readable on a laptop screen at full width (1200px-1920px)
- Mobile responsiveness is a bonus but not required
- Use consistent spacing and padding throughout
- Maintain visual hierarchy with clear sections

### Data Visualization
- **Charts must include:**
  - A clear, descriptive title
  - Labelled axes (where applicable)
  - A legend explaining data series
- **Chart styling:**
  - Do not use dark backgrounds for chart areas
  - Keep charts light and readable
  - Use brand colors consistently in visualizations
  - Ensure sufficient contrast for accessibility

### Content Guidelines
- **No emojis:** Keep the interface clean and professional—use icons instead if visual elements are needed
- **Use sentence case:** Capitalize only the first word and proper nouns in headings, titles, and labels—not each word
- Use clear, concise language
- Write descriptive alt text for any images or visualizations
- Organize content with clear headings and sections
