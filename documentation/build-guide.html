Build Guide

← Back to Overview

Welcome to the build phase! This is where your dashboard comes to life. You'll use GitHub Copilot to generate a single-page website that visualizes your data and tells a compelling story about conservation.

Don't worry if you're not a web developer—Copilot will be your coding partner. Your job is to guide it with clear instructions and iterate until you're proud of what you've built.

Before You Start

Make sure you have these three key assets ready. They're the foundation for building your dashboard:

Your Build Assets
  • CSV Files from your analysis phase (located in your output/ folder)
  • Dashboard Spec (the -spec.md file you exported from the Dashboard Builder)
  • Technical Requirements (found in TECHNICAL_REQUIREMENTS.md in your repo)
  • Your Own Design Guidelines (create a simple markdown file with your team's color palette, fonts, and style preferences)

These files tell Copilot what to build (your spec), how to build it (technical requirements), and how it should look (your design guidelines).

Create Your Design Guidelines:

Before you start coding, spend 10 minutes as a team to create a simple DESIGN_GUIDELINES.md file. Include: your color palette (3-5 colors with hex codes), font choices, and any style preferences (modern, playful, professional, etc.). This gives your dashboard a consistent, polished look and makes it easier to prompt Copilot.

Getting Started with Copilot

Think of GitHub Copilot as your AI pair programmer. You describe what you want in natural language, and Copilot generates the code. Here's how to work with it effectively:

1

Open Chat

In VS Code, open Copilot Chat (Ctrl+Shift+I or Cmd+Shift+I). This is where you'll have conversations with Copilot about your code.

2

Reference Files

Use #file to reference your files. For example: #file:dashboard-spec.md or #file:TECHNICAL_REQUIREMENTS.md

3

Be Specific

The more detail you provide, the better. Instead of "make a chart," say "create a bar chart showing forest loss by region using data from forest-loss.csv"

4

Iterate

Don't expect perfection on the first try. Review what Copilot creates, then ask it to refine: "make the colors match our brand" or "add labels to the axes"

Your First Prompt

Here's a starter prompt to kick off your dashboard. Copy this into Copilot Chat and customize it with your specifics:

Example Starter Prompt

"I need to create a single-page HTML dashboard for my WWF Discovery Day project. Please use #file:dashboard-spec.md as the product specification, #file:TECHNICAL_REQUIREMENTS.md for technical guidance, and #file:DESIGN_GUIDELINES.md for styling.

The dashboard should have a header with the title '[Your Dashboard Title]', an introduction section, and a 3×3 grid of charts as specified in my dashboard spec.

Use the CSV files in my output/ folder for the data. Start by creating the HTML structure with placeholder charts, and we'll add the real visualizations one by one."

Pro Tip:

Start simple. Get the basic structure working first (header, intro, grid layout), then add charts one at a time. This makes debugging easier and gives you quick wins.

Building Your Charts

Once you have the basic structure, it's time to bring your data to life with visualizations. Here's how to work with Copilot to create each chart:

For Each Chart
  • Tell Copilot which CSV file to use
  • Specify the chart type (bar, line, pie, etc.)
  • Describe what data should be on each axis
  • Ask for labels, legends, and tooltips
  • Request your team's brand colors from your design guidelines
Example Chart Prompt

"Create a bar chart in the first grid cell using data from output/forest-loss.csv. Show forest loss in hectares by region. Use the primary color from my design guidelines for bars. Include a title 'Forest Loss by Region 2020-2024', labeled axes, and a legend."

Copilot will likely use a charting library like Chart.js or Plotly. Don't worry about the technical details—just review the output and ask for changes if needed.

Stuck?

If a chart isn't working, ask Copilot: "The chart isn't displaying. Can you debug this?" or "The data isn't loading from the CSV. What's wrong?" Copilot can troubleshoot its own code!

Polish and Iterate

Once your dashboard is functional, it's time to make it shine. Here are some ways to take it to the next level:

  • Add interactivity: "Make the charts interactive so users can hover to see exact values"
  • Improve layout: "Add more spacing between the grid cells" or "Make the header image full-width"
  • Enhance text: "Below each chart, add the summary text from my dashboard spec"
  • Refine colors: "Use a gradient for the background matching the purple theme"
  • Optimize mobile: "Make the grid single-column on mobile screens"
  • Add animations: "Animate the charts when the page loads"
Get Creative:

This is your chance to experiment! Try asking Copilot for things like: "Add a dark mode toggle," "Create a scrolling reveal effect," or "Add icons next to section headers." The worst that can happen is it doesn't work—and you can always undo.

Preview Your Dashboard

As you build, you'll want to see your changes in action. Here's how to preview your dashboard locally:

Preview Steps
  • Save your index.html file in VS Code
  • Right-click on the file in the file explorer
  • Select "Copy Path" to copy the full file path
  • Open your web browser (Chrome, Firefox, Safari, etc.)
  • Paste the file path into the address bar and press Enter
  • Your dashboard will load locally in the browser
Refresh to See Changes:

After making changes in VS Code, save the file and refresh your browser (Cmd+R or Ctrl+R) to see the updates. Keep both VS Code and your browser visible side-by-side for the fastest iteration.

Final Tips

Work as a Team:

Divide and conquer! One person can work on the layout, another on charts, and another on polishing text. Use Copilot in parallel and merge your work.

Ask Your Coach:

Your coach is there to help! If you're stuck, show them what Copilot generated and ask for feedback. They can help you refine your prompts.

Have Fun:

This is your chance to build something real that highlights important conservation work. Be proud of what you create, learn from the process, and enjoy the magic of AI-assisted coding!
