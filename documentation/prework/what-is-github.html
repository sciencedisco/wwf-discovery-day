What is GitHub?

GitHub: Your team workspace
GitHub is a platform where teams store, track, and collaborate on files—especially code and data.
Think of it as a shared folder with superpowers

Track Changes
Collaborate
Repository = Your project folder
A "repo" is just a folder containing all your project files, plus a history of every change made.

Branch = Your personal workspace
Create a branch to work on your part without affecting others. It's like having your own copy to experiment in.

Best Practice: Name your branch after yourself (e.g., "sarah-visualizations") so teammates know who's working on what.

Changes exist only on your computer until you commit and push them!

Commit = Save a checkpoint
A "commit" is like saving a checkpoint in a video game. It captures your changes with a description.
VS Code: Source Control
Changes (3)

data.json
Message: "Added deforestation visualization"
Commit
Commit Message Tip: Write what you did, like "Fixed chart colors" or "Added new data source"
Push = Upload to GitHub
"Pushing" uploads your commits from your computer back to GitHub so others can see them.

Pull request = "I'm done, please review"
When you're ready to submit your work, create a Pull Request. It's like saying "Here's my contribution, ready for feedback!"
